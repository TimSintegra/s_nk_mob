# Спецификация: Импорт дерева работ из Excel файла "Структура ЕР.xlsx"

## 1. Общая структура файла

Файл `data/Структура ЕР.xlsx` содержит **7 листов**, каждый из которых является **отдельным корневым деревом** на сайте:

| # | Имя листа | Корневой узел (H2) | Описание |
|---|-----------|-------------------|----------|
| 1 | CS00 КАБЕЛЕНЕСУЩИЕ КОНСТРУКЦИИ, | CS00 | Кабеленесущие конструкции, кабелепроводы и конструкции под оборудование |
| 2 | II00 МОНТАЖ ОБОРУДОВАНИЯ КИПиА | II00 | Монтаж оборудования КИПиА |
| 3 | EL00 ЭЛЕКТРОМОНТАЖНЫЕ РАБОТЫ | EL00 | Электромонтажные работы |
| 4 | EL II Монтаж подключение расклю | EL/II | Монтаж/подключение/расключение кабеля |
| 5 | DW00 ДЕМОНТАЖНЫЕ РАБОТЫ | DW00 | Демонтажные работы |
| 6 | ET00 СИСТЕМА ЭЛЕКТРООБОГРЕВА | ET00 | Система электрообогрева |
| 7 | HU00 ПЕРЕХОДЫ ПОДЗЕМНЫЕ | HU00 | Переходы подземные |

**Важно:** EL00 и EL II — это **разные вещи** (разные корневые деревья).

---

## 2. Структура дерева (иерархия уровней)

### 2.1. Общая схема

```
Уровень 0: Корень (из строки 1 листа)
├── Уровень 1: Категория (из строки 2 листа)
│   ├── Уровень 2: Подкатегория (код с 1 тире, например EL00-08)
│   │   ├── Уровень 3: Подподкатегория (код с 2 тире, например EL00-08-02)
│   │   │   ├── Уровень 4: Вариант (без кода, например "1-жильный") — ТОЛЬКО в листе EL II
│   │   │   │   ├── Уровень 5: Элемент (код с 3+ тире, например EL00-08-02-60)
│   │   │   │   └── Уровень 5: Элемент (код с 3+ тире, например EL00-08-02-61)
│   │   │   ├── Уровень 4: Вариант (например "3-жильный")
│   │   │   │   └── Уровень 5: Элемент
│   │   │   └── Уровень 4: Элемент (если вариантов нет)
│   │   └── Уровень 3: Элемент (если подподкатегорий нет)
│   └── Уровень 2: Элемент (если подкатегорий нет)
└── ...
```

### 2.2. Правила определения уровня

1. **Код узла определяет уровень** — количество тире в коде = глубина вложенности
2. **Корень** (уровень 0): Имя из строки 1, например "EL/II" (H2 тег), с описанием "Монтаж/подключение/расключение кабеля"
3. **Категория** (уровень 1): Из строки 2 (без кода), например "Кабель питания и управления"
4. **Подкатегория** (уровень 2): Код с 1 тире, например "EL00-08"
5. **Подподкатегория** (уровень 3): Код с 2 тире, например "EL00-08-02"
6. **Вариант** (уровень 4, ТОЛЬКО EL II): Без кода, например "1-жильный", "3-жильный", "1-4 жил"
7. **Элемент** (уровень 5+): Код с 3+ тире, например "EL00-08-02-60"

### 2.3. Исключения

- **Не все листы имеют уровень "вариант"** — только EL II
- **Максимальная глубина может быть больше 6 уровней** в некоторых листах
- **Строки определяются по содержимому** ячеек, а не по фиксированным номерам строк

## 3. Формат данных в Excel

### 3.1. Структура строк (общая схема)

| Строка | Содержимое | Уровень дерева |
|--------|-----------|----------------|
| 1 | Название листа (корень) | Уровень 0: Корень |
| 2 | Категории (текст без кода) | Уровень 1: Категория |
| 3 | Подкатегории (код с 1 тире) | Уровень 2: Подкатегория |
| 4 | Типы работ (код с 2 тире) или варианты | Уровень 3: Подподкатегория или Вариант |
| 5 | Единицы измерения (метр, шт) или варианты | Атрибут или Уровень 4: Вариант |
| 6+ | Конкретные работы (код с 3+ тире) | Уровень 4-5: Элемент |

**Важно:** Строки определяются **по содержимому ячеек**, а не по фиксированным номерам. В разных листах структура может отличаться.

### 3.2. Пример (лист EL II)

```
Строка 1: [1] "EL/II Монтаж/подключение/расключение кабеля"  → Корень
Строка 2: [1] "Кабель питания и управления"                   → Категория
          [19] "Кабель контрольный"                            → Категория
          [37] "Кабель оптический/коаксиальный/специальный"   → Категория
Строка 3: [1] "EL00-08 Надземная прокладка"                   → Подкатегория
          [7] "EL00-16 Прокладка кабеля под землей"           → Подкатегория
          [13] "EL00-09 Работы по подключению кабелей"        → Подкатегория
Строка 4: [1] "1. EL00-08-02 Монтаж кабеля НИЗКОГО напряжения" → Подподкатегория
Строка 5: [1] "метр"                                          → Ед. изм. (атрибут)
Строка 6: [1] "1-жильный"                                     → Вариант
Строка 7: [1] "EL00-08-02-60 Монтаж кабеля НИЗКОГО напряжения, 1-жильный, сечением до 16 мм2" → Элемент
```

### 3.3. Пример (лист CS00 — без вариантов)

```
Строка 1: [1] "CS00 КАБЕЛЕНЕСУЩИЕ КОНСТРУКЦИИ, КАБЕЛЕПРОВОДЫ И КОНСТРУКЦИИ ПОД ОБОРУДОВАНИЕ"  → Корень
Строка 2: [1] "Монтаж лотков/коробов/..."                            → Категория
          [40] "Установка труб и кабельные проходки"                 → Категория
Строка 3: [1] "CS00-01 Монтаж лотков без установки опор"            → Подкатегория
          [9] "CS00-02 Монтаж лотков с установкой опор"             → Подкатегория
Строка 4: [1] "1. CS00-01-01 Лестничные лотки"                      → Подподкатегория
Строка 5: [1] "метр"                                                → Ед. изм. (атрибут)
Строка 6: [1] "1.1 CS00-01-01-13 Монтаж лестничного лотка ширина до 100 мм" → Элемент
```

## 4. Правила разбора

### 4.1. Определение корня

- Берём текст из ячейки A1 листа
- Разделяем на **заголовок** (H2 тег) и **описание**
- Пример: "EL/II" → заголовок, "Монтаж/подключение/расключение кабеля" → описание

### 4.2. Определение категорий (уровень 1)

- Берём из **строки 2** листа
- Каждая непустая ячейка в строке 2 — это категория
- Категории **не имеют кода** — это просто текст

### 4.3. Определение подкатегорий (уровень 2)

- Берём из **строки 3** листа
- Каждая непустая ячейка в строке 3 — это подкатегория
- Подкатегории **имеют код с 1 тире** (например, "EL00-08", "CS00-01")
- Формат: "КОД Название", например "EL00-08 Надземная прокладка"

### 4.4. Определение подподкатегорий (уровень 3)

- Берём из **строки 4** листа
- Каждая непустая ячейка в строке 4 — это подподкатегория
- Подподкатегории **имеют код с 2 тире** (например, "EL00-08-02", "CS00-01-01")
- Формат: "1. КОД Название", например "1. EL00-08-02 Монтаж кабеля НИЗКОГО напряжения"

### 4.5. Определение вариантов (уровень 4, ТОЛЬКО EL II)

- Варианты **не имеют кода** — это просто текст
- Примеры: "1-жильный", "3-жильный", "1-4 жил", "5-8 жил"
- Варианты появляются **после единицы измерения** в столбце
- Варианты **только в листе EL II** — в других листах их нет

### 4.6. Определение единиц измерения (атрибут)

- Единицы измерения **не являются узлами дерева**
- Это **атрибуты** (свойства) элементов
- Формат: "метр", "шт", "тонна", "м2", "кг" и т.д.
- Единица измерения применяется ко **всем элементам ниже** в столбце (до следующей единицы измерения)
- Единица измерения из строки 5 применяется к элементам в строках 6+

### 4.7. Определение элементов (уровень 4-5)

- Элементы **имеют код с 3+ тире** (например, "EL00-08-02-60", "CS00-01-01-13")
- Формат: "КОД Название", например "EL00-08-02-60 Монтаж кабеля НИЗКОГО напряжения, 1-жильный, сечением до 16 мм2"
- Элементы — это **конечные работы**, которые выбирают для отчёта

## 5. Привязка узлов по столбцам

### 5.1. Понятие "диапазона" (band)

Каждый столбец в Excel принадлежит определённому "диапазону" — группе столбцов, объединённых общим родителем.

**Правило:** Столбец принадлежит диапазону, начинающемуся в **ближайшем столбце ≤ текущего**.

**Пример (EL II):**
```
Столбцы 1-6:   Диапазон "Кабель питания и управления" (столбец 1)
Столбцы 7-12:  Диапазон "EL00-16 Прокладка кабеля под землей" (столбец 7)
Столбцы 13-18: Диапазон "EL00-09 Работы по подключению кабелей" (столбец 13)
Столбцы 19-24: Диапазон "Кабель контрольный" (столбец 19)
...
```

### 5.2. Привязка родителя

Каждый узел привязывается к родителю **в пределах своего диапазона**:

1. Если в диапазоне есть **вариант** (1-жильный) — родитель = вариант
2. Если в диапазоне есть **группа** (код с 2 тире) — родитель = группа
3. Если нет ни варианта, ни группы — родитель = **подкатегория** (из строки 3)
4. Если нет подкатегории — родитель = **категория** (из строки 2)
5. Если нет категории — родитель = **корень**

## 6. Модель данных (база данных)

### 6.1. Таблица WorkNode

```sql
CREATE TABLE work_node (
    id          SERIAL PRIMARY KEY,
    source_key  VARCHAR(255) UNIQUE NOT NULL,  -- Уникальный ключ: "Lист:Ячейка"
    code        VARCHAR(100),                   -- Код работы (EL00-08-02-60)
    name        VARCHAR(700),                   -- Название работы
    parent_id   INTEGER REFERENCES work_node(id), -- Родительский узел
    unit        VARCHAR(50),                    -- Ед. измерения (атрибут)
    sort_order  INTEGER DEFAULT 0,              -- Порядок (как в Excel)
    is_active   BOOLEAN DEFAULT TRUE            -- Активен ли узел
);
```

### 6.2. Поля

| Поле | Описание | Пример |
|------|----------|--------|
| `source_key` | Уникальный ключ узла (лист:ячейка) | "EL_II:A1", "EL_II:C7" |
| `code` | Код работы (если есть) | "EL00-08-02-60" |
| `name` | Название работы | "Монтаж кабеля НИЗКОГО напряжения, 1-жильный, сечением до 16 мм2" |
| `parent_id` | Ссылка на родительский узел | ID узла "EL00-08-02" |
| `unit` | Ед. измерения (только для элементов) | "метр" |
| `sort_order` | Порядок сортировки (как в Excel) | 1, 2, 3... |
| `is_active` | Активен ли узел | TRUE/FALSE |

### 6.3. Уникальность

- `source_key` — **единственный уникальный идентификатор** (формат: "Лист:Ячейка")
- `code` — **НЕ уникален** (один код может повторяться даже в пределах одного листа!)
- Для уникальной идентификации узла по коду нужно учитывать **родительский узел** или **лист**
- **ВАЖНО:** Поскольку коды могут повторяться, `source_key` — единственный способ однозначно идентифицировать узел

## 7. Пользовательский интерфейс

### 7.1. Интерактивное дерево

- Пользователь видит **7 корневых узлов** (по одному на лист)
- Может **раскрывать ветки** дерева
- Выбирает **конкретную работу** (элемент) для отчёта

### 7.2. Страница отчёта

После выбора работы пользователь заполняет:

| Поле | Описание |
|------|----------|
| Дата | Дата выполнения работы |
| Объект | Объект (стройка) |
| Количество выполненного объёма | Число + **ед. измерения из атрибута узла** |
| Титул | Титул проекта |
| Раздел проекта | Раздел проекта |
| Комментарий | Комментарий |

**Важно:** Единица измерения подставляется **автоматически** из атрибута выбранного элемента.

## 8. Требования к скрипту импорта

### 8.1. Входные данные

- Файл: `data/Структура ЕР.xlsx`
- Формат: Excel (.xlsx)
- Библиотека: openpyxl

### 8.2. Выходные данные

- Таблица `WorkNode` в базе данных PostgreSQL
- Флаг `--clear`: при импорте все существующие узлы помечаются как `is_active=False`

### 8.3. Логика работы

1. **Открыть файл** и перебрать все листы
2. **Для каждого листа:**
   a. Определить корень (строка 1)
   b. Определить категории (строка 2)
   c. Определить подкатегории (строка 3, код с 1 тире)
   d. Определить подподкатегории (строка 4, код с 2 тире)
   e. Определить варианты (без кода, только EL II)
   f. Определить единицы измерения (атрибуты)
   g. Определить элементы (код с 3+ тире)
   h. Создать узлы и установить связи parent-child
3. **Сохранить в базу данных** с уникальным source_key

### 8.4. Особенности

- **Строки определяются по содержимому** ячеек, а не по фиксированным номерам
- **Код узла определяет уровень** — количество тире = глубина
- **Порядок узлов** = порядок в Excel (сверху вниз, слева направо)
- **Единицы измерения** — атрибуты, не узлы
- **Варианты** — только в листе EL II
- **Merged cells** — содержат заголовки родительских узлов (объединяют столбцы)
- **Коды НЕ уникальны** — могут повторяться даже в пределах одного листа! Используйте `source_key` как единственный уникальный идентификатор

## 10. Отличия от текущего скрипта

| Проблема | Текущий скрипт | Нужно |
|----------|---------------|-------|
| Определение строк | Фиксированные строки 1-3 | По содержимому ячеек |
| Уровни дерева | 2 уровня (blue/green) | 5-7 уровней (по коду) |
| Варианты | Не обрабатываются | Обрабатываются (только EL II) |
| Ед. измерения | Создаёт узлы | Только атрибуты |
| Привязка родителя | По позиции столбца | По диапазону + варианту |
| Код узла | Извлекается, но не используется для уровней | Определяет уровень |

## 11. Вопросы для уточнения

1. **EL00 vs EL II**: Лист EL00 — это отдельные электромонтажные работы, а EL II — монтаж/подключение кабеля. Это правильно?
2. **Варианты**: Варианты (1-жильный, 3-жильный) появляются только после единицы измерения в столбце. Это верно?
3. **Merged cells**: Объединённые ячейки содержат заголовки родительских узлов (объединяют столбцы). Это правильно?
4. **Максимальная глубина**: В некоторых листах может быть больше 6 уровней. Это верно?
